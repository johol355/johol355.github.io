1 Introduction

In this markdown we will explore prediction of a patient being transferred by HEMS. These models are done with r packages such as and . There is no missing data handling. The Bayesian implementation with missing data handling has essentially the same results and has better convergence. But I need to add some tweaks before it is presentable, thus we will do frequentist stuff here.

2 Question 1: What patient level factors predict the utilization of HEMS-IFT in neuro-ICU patients?

This analysis is limited to patients transferred from centers that utilize HEMS-IFT in more than 20% of cases. Note that all predictors are patient level predictors, eg. physiologic parameters.

2.1 Univariate analysis of reasonable predictors

Here a set of predictors will be used in univariate logistic models to see if anything surprising pops out. Nothing shocking pops out.

2.2 Multivariate (varying intercept) analysis of reasonable predictors + strong univariate predictors

Here a set of predictors will be choosen based on domain knowledge and on the p-value for predictors being <.20 in the univariate analysis. Out of “markers of respiratory instability” and “ARDS” I’ve only choosen the first as the missingness is lower. This model is a varying intercept model where sending ICU is a level. The slopes are not varying per sending ICU. (I’ve triend a varying slope model in the Bayesian framework, but it is not much different.) Important predictors are weather conditions being acceptable and the patient being on mechanical ventilation. Transfers during summer also increase the chance of HEMS transfer. This could be due to staffing factors. But, I think it is mainly and effect of my HEMS minima predictor not telling the full story as it is only describing the weather conditions at the time of ICU discharge at the sending hospital (not the conditions expected in flight et cetera). Other than that, there is a trend sicker (as per SAPS score) being transferred by HEMS.

3 Question 2: What hospital level factors predict the utilization of HEMS-IFT in neuro-ICU patients?

Interesting predictors here would be thing such as: - Transfer distance (mean) - Having a helipad on the hospital - Having an airport nearby - The type of hospital

I’m not sure what type of model should be used to estimate this. I’ve been thinking of a count model, such as a Poisson model, but the issue is that the hospitals differ in size (and thus the number of trials at “a success”).

Where L is a set of measured confounders: - Diagnosis - Age - Sex - Sending site - SAPS score - Conciousness level - Presence of mechanical ventilation

Month is also a potential confounder (as it might affect staffing and the quality of ICU care, as well as the availability of HEMS, mainly via the weather exposure).

HEMS minima affects the exposure and is, conditional on the time of year, an instrumental variable that could potentially be used to infer the causal effect of HEMS-IFT. I’ve run into some issues here: (1) the “standard” way of using instrumental variables in a two-stage least squares regression is not applicable here, since the outcome is binary (this means that the odds-ratios are very hard to interpret/unusable due to the “non-collapsability” of this statistic); (2) It makes sense to do the analysis in a hierarchical manner, but the available software does not allow for this.

This is a quick test with the package doing IV analysis by way of Robins’/Hernan “g-estimation”.

Code 
library(ivtools)
fitX.L <- glm(formula=hems_ift ~ 1 + formatted_icu_name + DX_GROUP + age + SAPS_score_excluding_gcs_age + sir_consciousness_level + any_AMV + MONTH, data = d, family = binomial)
fitX.LZ <- glm(formula=hems_ift ~ 1 + formatted_icu_name + DX_GROUP + age + SAPS_score_excluding_gcs_age + sir_consciousness_level + any_AMV + MONTH + hems_minima, data = d, family = binomial)
fitY.LZX <- glm(formula=d30 ~ 1 + formatted_icu_name + DX_GROUP + age + SAPS_score_excluding_gcs_age + sir_consciousness_level + any_AMV + MONTH + hems_minima + hems_ift, data = d, family = binomial)

ivm <- ivglm(estmethod = "g", fitX.LZ = fitX.LZ, fitX.L = fitX.L, fitY.LZX = fitY.LZX, fitY.LX = fitY.LX, data = d, link="logit")

The outcome here is 30-day mortality. The point estimate for the coefficient for the causal predictor HEMS is -0.5266, i.e. OR = 0.59. The 95% CI would be:

Code 
exp(confint(ivm))
             2.5 %   97.5 %
hems_ift 0.1829341 1.906955

I.e., OR 0.18-1.9. Wide, that is. If 1-yr mortality is used, the OR point estimate is very close to being “signficant”. It could be reasonable to try out a time-to-event model as 30-day mortality is completely arbitrary.

What I don’t like is that this model uses sending ICU as a predictor rather than a level. Also, there is no missing data handling as always with these frequentist approaches. However, I have not found an obvious way to do this analysis in a Bayesian way yet.
